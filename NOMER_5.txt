Fungsi registerMember() berfungsi memeriksa apakah email yang diposting sudah ada. Jika demikian, ini memotong aliran pendaftaran dan mengembalikan kesalahan. Jika tidak, ini membuat kueri Sisipkan untuk menambahkan catatan anggota ke dalam database MySQL.
Fungsi loginMember() memeriksa apakah ada kecocokan untuk detail login yang dimasukkan. Jika kecocokan ditemukan, itu menghapus otentikasi dan memungkinkan pengguna untuk mengakses dasbor.
Fungsi empty() akan menghasilkan nilai false jika sebuah variabel telah diisi, dan bernilai true jika variabel tersebut belum isi.    Dengan menggunakan struktur IF dan fungsi empty(), kita bisa membuat logika validasi objek form mana saja yang dianggap perlu(harus diisi) dan mana yang boleh dikosongkan. Formulir - Kita perlu menggunakan atribut action dan post. Atribut tindakan akan disetel ke file otentikasi. Saat formulir dikirimkan, data formulir akan dikirim ke file otentikasi untuk diproses. Selain itu, metode ini dideklarasikan sebagai posting karena ini akan memungkinkan kita untuk memproses data formulir menggunakan metode permintaan POST.
- Input (teks / kata sandi) - Kita perlu memberi nama bidang formulir kita sehingga server dapat mengenalinya. Nilai nama atribut dapat kita deklarasikan sebagai username, yang dapat kita gunakan untuk mengambil variabel post di file otentikasi kita untuk mendapatkan datanya, misalnya: $ _POST ['username'].
- Input (kirim) - Saat pengiriman formulir, data akan dikirim ke file otentikasi kami untuk diproses.
